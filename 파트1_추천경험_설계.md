# 1번 파트: 추천 경험 담당

담당자: 최은택

## 담당 범위

1번 파트는 다음 기능만 책임집니다.

- 기분 입력
- 주종 입력
- 도수 입력
- 취향·향 입력
- 예산 입력
- 맞춤 주류 추천
- 추천한 이유와 쉬운 설명
- 다른 주류 추천

페어링, 음악, 찜·기록, 배포 기능은 다른 파트가 담당합니다. 현재 `app.py`에서는 기존 실행을 위해 함께 보일 수 있지만, 이 브랜치의 핵심 변경은 입력·추천·대체 추천 영역입니다.

## 입문자를 위한 질문 원칙

- 전문 용어보다 일상적인 표현을 사용합니다.
- 도수는 숫자를 바로 보여주기보다 `낮은 / 적당 / 높은`으로 먼저 질문합니다.
- 한 번에 너무 많은 선택지를 요구하지 않습니다.
- 추천 결과에는 제품명만 쓰지 않고, 왜 어울리는지 한 문단으로 설명합니다.
- 처음부터 개성 강한 술만 추천하지 않고 `입문 추천` 표시를 제공합니다.

## 입력 데이터 구조

```text
occasion: 혼술 / 친구들과 / 가족들과 / 연인과 / 기타
drink_type: 전체 / 와인 / 위스키 / 샴페인
mood: 차분하고 조용한 / 기분 좋은 / 위로받는
proof: 낮은 / 적당 / 높은
flavors: 달달한, 탄산감있는, 드라이한, 오크향, 오렌지, 훈연, 꿀
budget: 저가 / 중간 / 고가 / 프리미엄
```

## 추천 점수

```text
함께 마시는 사람 일치: 3점
주종 일치: 3점
분위기 일치: 3점
도수 일치: 2점
맛·향 하나 일치: 2점
예산 일치: 2점
```

점수가 높은 순서로 최대 3개를 보여줍니다. `다른 주류도 추천받기`를 누르면 앞서 본 술을 제외하고 다음 후보를 보여줍니다.

## 다른 파트와 연결할 데이터

다른 파트가 페어링이나 음악을 연결할 때 사용할 값은 다음과 같습니다.

```python
drink["name"]  # 술 이름
drink["type"]  # 와인, 위스키, 샴페인
drink["price"] # 가격대
drink["flavors"] # 맛과 향
drink["mood"] # 어울리는 분위기
```

이 키 이름은 변경하지 않는 것을 권장합니다. 키 이름이 바뀌면 페어링·음악 파트와 합칠 때 오류가 생길 수 있습니다.

## 이 브랜치 테스트 방법

```powershell
python -m py_compile app.py
streamlit run app.py
```

확인할 항목:

- 주종을 위스키로 선택하면 위스키가 우선 추천되는가
- 도수를 바꾸면 결과가 달라지는가
- 맛을 여러 개 선택할 수 있는가
- 추천 이유가 제품별로 다르게 표시되는가
- `다른 주류도 추천받기`를 누르면 이전 결과와 다른 술이 나오는가
